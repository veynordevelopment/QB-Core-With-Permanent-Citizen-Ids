## What changed

This version has been cleaned up and adjusted so it feels more polished and consistent.

### Included changes
- Removed unnecessary readme / template / extra clutter files
- Switched the visible ID wording over to **Passport ID**
- Cleaned up the shared/item/admin handling for Passport ID usage
- Kept important compatibility in place so it still works smoothly with existing QB setups
- Improved the overall structure so it is easier to use and maintain

### Compatibility note
Under the hood, some QB systems still rely on existing identifier logic.  
This version was adjusted carefully so the resource can use **Passport ID** in the places that matter without causing avoidable breakage.
